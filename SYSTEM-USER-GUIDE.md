## 1. システム概要

### 1.1 システムの目的

LNポータルは、**発信者情報開示請求・裁判業務を効率化**するためのオールインワンシステムです。

**主な機能**:
- 📊 案件（LN）の一元管理
- 📝 申立書の自動生成
- 📚 業務マニュアルの提供
- 🎮 実績・レベル管理（ゲーミフィケーション）
- 💬 フィードバック・改善提案

### 1.2 対象ユーザー

| ロール | 名称 | 主な業務 |
|--------|------|----------|
| **beginner** | 初級者 | 証拠収集、LNチェック |
| **expert** | 中級者 | 申立書作成、書類詰め |
| **leader** | 上級者 | 全体管理、発送、指導 |

### 1.3 システムの特徴

✅ **オフライン動作**: 外部ファイル・ネットワーク不要  
✅ **シングルページ**: HTML1ファイルで完結  
✅ **ローカルストレージ**: ブラウザにデータ保存  
✅ **レスポンシブ**: PC・タブレット・スマホ対応  
✅ **リアルタイム更新**: 自動保存・即時反映

## 5. LN一覧画面

### 5.1 画面構成

```
┌──────────────────────────────────────┐
│  フィルタタブ                        │
│  [全LN 13件] [今日やること 8件]    │
└──────────────────────────────────────┘
┌──────────────────────────────────────┐
│  検索バー                            │
│  [訴番・申立人・相手方で検索...]    │
└──────────────────────────────────────┘
┌──────────────────────────────────────┐
│  LNカード一覧                        │
│  ┌──────────────────────────────┐  │
│  │ 7K14488  申立人: ●●            │  │
│  │ 相手方: ▲▲                    │  │
│  │ [証拠] ○○○○  [書類] ●●●    │  │
│  │ [詳細を見る ▼]                 │  │
│  └──────────────────────────────┘  │
└──────────────────────────────────────┘
```

### 5.2 フィルタタブ（P-22〜P-25）

**全LN（P-22）**:
- すべての案件を表示

**今日やること（P-23）**:
- 本日対応が必要な案件のみ

**優先（P-24）**:
- 優先度が高い案件のみ

**書類未完（P-25）**:
- 書類が揃っていない案件のみ

### 5.3 LNカードの見方

#### 基本情報
```
┌───────────────────────────────────┐
│ 7K14488                           │ ← 訴番
│ 申立人: ●●●●                     │ ← 申立人名
│ 相手方: ▲▲▲▲                     │ ← 相手方名
│ 🟢 準備完了                       │ ← ステータス
└───────────────────────────────────┘
```

#### ステータスの色分け

🟢 **緑（準備完了）**:
- 証拠収集完了
- 申立書作成可能

🟡 **黄色（作業中）**:
- 証拠収集中
- 一部未完

🔴 **赤（未着手）**:
- まだ作業開始していない
- 早急な対応が必要

### 5.4 証拠進捗の表示

クリックして展開すると、詳細な証拠進捗が表示される：

```
証拠進捗（A 証拠収集）:
✅ 受任通知
✅ プロバイダ開示
✅ 不開示
✅ 甲10（URL一覧）
✅ 甲11（削除済み画面）
⬜ 甲13（DVDパッケージ）
⬜ 甲14（確認書）
```

**記号の意味**:
- ✅: 完了
- ⬜: 未完了
- ⚠️: 要確認

### 5.5 書類把握タブ（P-21）

**表示内容**:
- 受任通知: ○ / ×
- プロバイダ開示: ○ / ×
- 不開示: ○ / ×
- 甲13（DVDパッケージ）: ○ / ×
- 甲14（確認書）: ○ / ×

**一括更新**:
- チェックボックスをクリックで更新
- リアルタイムで保存

### 5.6 LNカードの展開機能

**展開すると表示される情報**:

1. **詳細な証拠進捗**
   - A 証拠収集: ●/●件完了
   - B 申立書作成: ●/●件完了

2. **アクション**
   - 📝 makelnで開く（P-30）
   - 📁 Drive: LNフォルダ（P-31）
   - 📁 Drive: plフォルダ（P-32）
   - 🗄️ DBファイル（開発中）（P-33）

3. **メモ欄**
   - 自由記述
   - 自動保存

### 5.7 まとめて作成依頼（P-29）

**機能**:
- 書類未完のLNを一括で抽出
- Google Chatにコピペできるテキストを生成

**使い方**:
1. ボタンをクリック
2. 生成されたテキストをコピー
3. Google Chatに貼り付け

**生成例**:
```
【書類未完LN一覧】
・7K14488: 甲13, 甲14 不足
・9S14492: 甲14 不足
・1K14196: 甲13 不足
```

## 6. 申立書作成（makeln）

### 6.1 アクセス方法

- サイドバー: **📝 書類作成（M-01）**
- または、LN一覧から **📝 makelnで開く**

### 6.2 画面構成

```
┌──────────────────────────────────────┐
│  案件情報                            │
│  訴番: 7K14488                       │
│  申立人: ●●●●                      │
│  相手方: ▲▲▲▲                      │
└──────────────────────────────────────┘
┌──────────────────────────────────────┐
│  申立書を作成する                    │
│  ┌────────────────────────────────┐ │
│  │ 申立ての種類                    │ │
│  │ ◉ 開示のみ                      │ │
│  │ ○ 開示＋消去禁止                │ │
│  │ ○ 開示＋提供命令                │ │
│  └────────────────────────────────┘ │
│  ┌────────────────────────────────┐ │
│  │ 著作権証明パターン              │ │
│  │ [-- 選択 --           ▼]       │ │
│  └────────────────────────────────┘ │
│  [📄 申立書を作成 M-10]             │
└──────────────────────────────────────┘
```

### 6.3 申立書本文の作成

#### 申立ての種類（ラジオボタン）

**開示のみ**:
- 発信者情報開示のみ請求
- 最も一般的なパターン

**開示＋消去禁止**:
- 発信者情報開示 + 侵害情報の削除禁止
- 削除される可能性がある場合

**開示＋提供命令**:
- 発信者情報開示 + ログ等の提供命令
- より強力な措置が必要な場合

#### 著作権証明パターン（ドロップダウン）

選択肢：
1. ① パッケージ著作表記
2. ② 映画製作者パターン（監督あり）
3. ③ パッケージ表記（監督なし）
4. ④ 職務著作
5. ⑤ 著作権譲渡
6. ⑥ 商標表記
7. ⑦ 販売サイト表記
8. ⑧ 倫理機構審査
9. ⑨ 複数監督
10. ⑩ 委託販売

**パターンの選び方**:
- 権利者の種類に応じて選択
- 迷ったら expert / leader に相談

#### 主張内容の入力

自由記述欄：
- 追加の主張を記載
- ●●●などのプレースホルダーを置き換える

### 6.4 申立書の生成（M-10）

**手順**:
1. 申立ての種類を選択
2. 著作権証明パターンを選択
3. 主張内容を入力（必要に応じて）
4. **📄 申立書を作成**ボタンをクリック
5. プレビューが表示される
6. 問題なければ、PDF / Markdown でダウンロード

**出力形式**:
- **PDF**: 印刷・提出用
- **Markdown**: 編集・修正用

### 6.5 証拠説明書の作成

**自動生成機能**:
1. **🔧 自動生成**ボタンをクリック
2. LNに紐づく証拠を自動抽出
3. 証拠説明書が生成される

**手動調整**:
- 証拠番号を修正
- 説明文を追加・変更
- 並び順を調整

**出力**:
- **📄 PDF**: 印刷用
- **📝 MD**: 編集用

### 6.6 別紙一式の作成

**包含する別紙**（チェックボックス）:
- ✅ 別紙1: 発信者情報目録
- ✅ 別紙2: IPN一覧（動画目録）
- ✅ 別紙3: 著作権関係証明

**機能**:
- チェックした別紙のみを出力
- 一括でPDF生成

### 6.7 著作権証明テンプレート集

**カテゴリ**:
- 権利者の問題
- 弁護士の問題
- 証拠記載テンプレート

**使い方**:
1. カテゴリを展開
2. 該当するテンプレートを探す
3. **コピー**ボタンで内容をコピー
4. 主張内容欄に貼り付け
5. ●部分を編集

**主なテンプレート**:
1. ① パッケージに著作表記
2. ② 販売サイト画像/パッケージ＋確認書
3. ③ hmp（簡易版）
4. ④ ベンハー（公式サイト著作表示）
5. ⑤ プレステージ（商号・商標表示）
6. ⑥ ホットエンターテイメント
7. ⑦ Cont（複数監督）
8. ⑧ エムケイエイチ（レーベル説明）
9. ⑨ ブルーコーナー（譲渡あり）
10. ⑩ KSプロ（委託販売）

### 6.8 テンプレートの追加（expert / leader のみ）

**手順**:
1. **＋ テンプレートを追加**をクリック
2. カテゴリを選択（複数選択可）
3. テンプレート名を入力
4. テンプレート本文を入力
5. **保存**をクリック

**カテゴリ**:
- 申立書
- 弁護士
- 権利者別
- 証拠
- その他

## 12. トラブルシューティング

### 12.1 システムが開かない

**原因**:
- ブラウザの互換性問題
- JavaScript無効

**対処法**:
1. ブラウザを最新版に更新
2. JavaScript を有効化
3. 別のブラウザで試す（Chrome推奨）

### 12.2 データが保存されない

**原因**:
- ローカルストレージの容量不足
- プライベートブラウジングモード

**対処法**:
1. ブラウザのキャッシュをクリア
2. 通常モードで開く
3. ストレージ容量を確認

### 12.3 ユーザーが選択できない

**原因**:
- データの読み込み失敗

**対処法**:
1. ページをリロード（⌘ + R）
2. ブラウザのコンソールでエラー確認

### 12.4 LNカードが展開しない

**原因**:
- JavaScriptエラー
- データ不整合

**対処法**:
1. ページをリロード
2. 別のLNで試す
3. ブラウザのコンソールを確認

### 12.5 申立書が生成できない

**原因**:
- 必須項目が未入力
- テンプレートの不備

**対処法**:
1. すべての項目を入力
2. 別のパターンを試す
3. expertに相談

## 13. よくある質問

### Q1: 複数のブラウザで使えますか？

**A**: はい、可能です。ただし、データはブラウザごとに別々に保存されます。

### Q2: オフラインで使えますか？

**A**: はい、完全にオフラインで動作します。ただし、初回アクセス時のみインターネット接続が必要です。

### Q3: データのバックアップは必要ですか？

**A**: 推奨します。ブラウザのキャッシュクリアでデータが消える可能性があります。

### Q4: ロールを変更できますか？

**A**: ユーザー選択ドロップダウンから別のユーザーを選ぶことで、別のロールを体験できます。

### Q5: フィードバックはどこに送信されますか？

**A**: ローカルストレージに保存され、フィードバック管理画面で確認できます。

### Q6: Google Driveのリンクはどこで設定しますか？

**A**: `db_case.js` ファイル内の `caseData` に直接記載します（開発者向け）。

### Q7: 新しいLNを追加するには？

**A**: `db_case.js` の `caseData` に新しいオブジェクトを追加します（開発者向け）。

### Q8: ポイントは手動で調整できますか？

**A**: `db_game.js` の `USER_GAME_DATA` を編集することで可能です（開発者向け）。

### Q9: システムは定期的に更新されますか？

**A**: はい、GitHub Pagesで公開されているバージョンは随時更新されます。

### Q10: マニュアルは印刷できますか？

**A**: はい、ブラウザの印刷機能（⌘ + P）で印刷できます。
