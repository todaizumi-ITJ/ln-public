1.はじめに — このシステムの使い方

1.1このシステムって何？全レベル

このシステムは「発信者情報開示請求」の書類作成を管理するツールです。

やることをひと言でいうと —

「違法アップロードした人の情報を、プロバイダ（通信会社）に教えてもらうための裁判所への申し立て」を準備する

著作権を持つ会社（申立人）の代理として、弁護士がプロバイダに開示請求をします。その書類作成・証拠集め・提出管理をこのシステムで行います。

難しそうに聞こえますが、手順は決まっているので、マニュアル通りにやれば大丈夫です。

1.24つの画面を知ろう全レベル

システムは4つの画面でできています。ブラウザで開くだけで使えます。

① ポータル
何ができる？ LN（事件）の一覧を見る、今日やることを確認する
いつ使う？ 朝一番にまず開く
→ lnportal.html
② 書類作成（makeln）
何ができる？ 申立書の作成、証拠チェック、Word出力
いつ使う？ 担当LNの書類を作るとき
→ lnmake.html
③ マニュアル（いまここ）
何ができる？ 用語・手順・チェックリストを確認
いつ使う？ やり方がわからないとき
④ 実績・レベル（game）
何ができる？ 自分のレベル・ポイント・チーム状況
いつ使う？ 成長の確認、日次レポート
→ lngame.html

各画面は独立しているので、どれかが壊れても他は影響を受けません。

1.3ロール別クイックスタート全レベル

あなたのロール（役割）によって、やることが違います。ポータル右上のユーザー切替で確認できます。

beginner — まずはここから

  1. ポータルを開く → ホーム画面のステップナビに従う
  2. 「LN一覧」で今日やること（Ready）のLNを確認
  3. LNをクリック → CSVダウンロードできるか、Driveにファイルがあるか確認
  4. 問題なければ「確認完了」ボタンで報告。不安なら先輩に聞く

一日の目安: LN確認 3〜5件 + 書類ボックス詰め

expert — 書類作成のメイン担当

  1. ポータルでLN一覧を確認 → 「すぐ作成可能」なLNを選ぶ
  2. makelnで開く → 「証拠を集める」タブで証拠チェック
  3. 「書類作成」タブでテンプレ選択 → プレビュー → Word出力
  4. 弁護士にチェック依頼 → USB作成 → 提出準備

一日の目安: 申立書作成 2〜4件 + 証拠確認 + beginnerサポート

leader — チーム管理＋実務

  1. ポータルのLN一覧 →「ステータスビュー」で全体の進捗確認
  2. 実績ページの日次レポートでメンバーの稼働チェック
  3. 委任状・証明書が揃ったLNから優先的に申立書作成を指示
  4. 例外案件の判断、beginnerの指導

ポイント: 委任状とKKが揃ったLNから着手するのが効率的

1.41つのLN（事件）ができるまで全レベル

LNは事件番号のこと。例: 7K14488。1つのLNが完了するまでの流れ:

① 登録expertが新規LN登録
→
② lo取得プロバイダからPDF取得
→
③ pl作成証拠PDFまとめ（自動）
→
④ 準備確認beginnerがCSV/Drive確認
→
⑤ 申立書作成expertがWord出力
→
⑥ 提出leader/expertが裁判所へ

beginnerさんは④がメイン。expertは②③⑤。leaderは⑥と全体管理。

LNデータ（caseData）の構造

基本情報: applicant、opponent、status、lawyer、ipCount、date
リンク: driveFolderUrl、plDriveUrl
自動処理: automationFeedback（処理状況メッセージ配列）
証拠進捗 ev: kk/pl/lo の done/total、kkMissing/plMissing、docs、manual

ファイル構成

lnmake/
├─ db_case.js ← メインDB（LN・ユーザー・テンプレ）
├─ db_game.js ← ゲームDB（レベル・ポイント）

└─ lngame.html ← 実績・レベル画面

2.証拠の集め方

2.1委任状の集め方全レベル

委任状は、申立人（権利者会社）から弁護士へ代理権を委任することを証明する書類です。

必要な情報

  • 申立人の正式名称 — 会社名、代表者名
  • 弁護士名 — 手続代理人となる弁護士全員の氏名
  • 事件名 — 発信者情報開示命令申立事件
  • 作成日 — 申立日から3ヶ月以内であることを確認

集め方の手順

  1. 申立人に連絡 — メールまたは電話で委任状の作成を依頼
  2. テンプレートを送付 — 弁護士事務所から提供されるテンプレートを使用
  3. 記入・押印を依頼 — 申立人側で記入・代表者印を押印してもらう
  4. 原本を郵送で受領 — PDFではなく原本が必要
  5. 内容確認 — 弁護士名、申立人名、日付が正しいか確認
  6. 有効期限チェック — 取得から3ヶ月以内であることを確認
注意点
  • 委任状は原本が必要（コピー不可）
  • 代表者印（会社実印）が押印されていること
  • 複数の弁護士が担当する場合、全員の氏名が記載されていること
  • 作成日から3ヶ月を超えると無効

申立人ごとに委任状の書式が異なる場合があるので、担当弁護士に確認してください。

2.2代表者事項証明書の集め方全レベル

代表者事項証明書は、会社の代表者が正式に登記されていることを証明する書類です。法務局で取得します。

必要な情報

  • 会社の正式名称 — 登記簿上の名称
  • 本店所在地
  • 代表者氏名
  • 会社法人等番号 — あると取得がスムーズ

取得方法（3つの方法）

① オンライン申請
登記・供託オンライン申請システムで申請。郵送または窓口受取
② 窓口申請
最寄りの法務局で申請書を記入して提出。即日発行
③ 郵送申請
申請書と返信用封筒を法務局に郵送

集め方の手順

  1. 申立人に確認 — 会社の正式名称、本店所在地を確認
  2. 取得方法を決定 — オンライン・窓口・郵送のいずれか
  3. 手数料を準備 — 1通600円（窓口は収入印紙、オンラインは電子納付）
  4. 申請書を作成 — 法務局の書式に従って記入
  5. 証明書を取得 — 発行されたら内容を確認
  6. 有効期限を確認 — 取得から3ヶ月以内
注意点
  • 取得から3ヶ月以内のものが有効
  • 代表者の氏名が正しく記載されているか確認
  • 古い証明書を使い回さないこと
  • 申立人が複数の場合、それぞれの証明書が必要

オンライン申請が便利です。登記・供託オンライン申請システム →

2.3甲1: USB 著作物の集め方expert

甲1号証（USB著作物）は、侵害された著作物そのものを証拠として提出するためのUSBメモリです。

必要なもの

  • 著作物の動画ファイル — 申立人が権利を持つ動画
  • USBメモリ — 新品または完全にフォーマットしたもの
  • ファイル名の命名規則 — LN番号とファイル番号を含める

作成手順

  1. 申立人から著作物を受領 — GoogleDrive、転送サービス、または直接受取
  2. ファイル形式を確認 — MP4、AVI、MOVなど一般的な形式
  3. USBを準備 — 容量は動画サイズに合わせる（通常8GB〜32GB）
  4. ファイルをコピー — 著作物をUSBのルートディレクトリにコピー
  5. ファイル名を確認 — 例: 7K14488_01.mp4
  6. 再生確認 — 正常に再生できることを確認
  7. ラベルを貼付 — USB本体にLN番号を記載
注意点
  • 著作物は申立人から正式に受領したもののみ使用
  • ファイルが破損していないか必ず再生確認
  • USBにウイルスチェックを実施
  • 裁判所提出用なので、余計なファイルを入れない
  • USBは書き込み可能な状態で提出（ロックしない）

複数の著作物がある場合は、すべて1つのUSBにまとめて入れます。ファイル名の連番で区別してください。

2.4別紙の種類と集め方expert

別紙は申立書に添付する一覧表です。以下の4種類があります。

4種類の別紙

① 当事者目録
申立人・相手方・代理人弁護士の情報
② 著作物目録
権利侵害された著作物の一覧
③ 動画目録
侵害動画のURL、タイトル、アップロード日時
④ 発信者情報目録
開示を求める情報（IPアドレス、タイムスタンプ等）

集め方・作成手順

① 当事者目録

  • 申立人の正式名称、代表者名、住所 — 登記簿謄本から転記
  • 相手方（プロバイダ）の情報 — 公式サイトから取得
  • 弁護士の氏名、事務所名、住所 — 委任状と一致させる

② 著作物目録

  • 著作物のタイトル、発行日、著作者名
  • DVD品番、ISBN等の識別番号
  • 申立人から提供された情報をもとに作成

③ 動画目録

  • 侵害動画のURL（証拠保全したもの）
  • 動画タイトル、再生時間
  • アップロード日時、発見日時
  • IPNデータベースから自動生成可能

④ 発信者情報目録

  • 発信者のIPアドレス
  • タイムスタンプ（通信日時）
  • ポート番号
  • プロバイダから提供されたloファイルから転記
注意点
  • すべての別紙に「別紙」と明記
  • ページ番号を振る
  • 申立書本文から「別紙〇〇参照」で参照される形式
  • 誤記がないか複数人でダブルチェック

書類作成画面（lnmake.html）の「申立書作成」タブで、別紙も自動生成できます。→ 書類作成画面へ

2.5書類ボックス詰めの手順全レベル

裁判所に提出する書類をボックスにまとめる作業です。順番と内容が重要です。

必要書類チェックリスト

  • 委任状（原本）
    申立人から受領した原本
  • 代表者事項証明書
    取得から3ヶ月以内
  • 申立書（正本・副本）
    正本1通、副本は相手方の数
  • 証拠説明書
    甲号証の一覧表
  • 別紙4種
    当事者・著作物・動画・発信者情報
  • 甲号証（甲1〜甲13）
    印刷物またはPDF（USB）
  • 収入印紙
    申立手数料（金額は案件による）
  • KK（DB記録印刷書面）
    申立人により不要の場合あり

ボックス詰めの順番

  1. 申立書（正本） — 一番上
  2. 証拠説明書
  3. 甲号証 — 甲1、甲2...の順番
  4. 別紙4種 — 当事者→著作物→動画→発信者情報の順
  5. 委任状
  6. 代表者事項証明書
  7. その他 — KK、追加資料等

確認ポイント

  • 各書類のページ数が揃っているか
  • ホチキス留めは左上1箇所
  • USB等の電子媒体は専用ケースに入れる
  • ボックスにLN番号を明記
  • 提出先裁判所・提出日を確認
よくあるミス
  • 委任状の有効期限切れ（3ヶ月超過）
  • 代表者事項証明書の有効期限切れ
  • 別紙の順番が違う
  • 甲号証の欠落（甲5が抜けている等）
  • 申立書と別紙の内容不一致
0 / 8 件完了

提出前に必ずexpertまたはleaderによる最終確認を受けてください。

2.6証拠説明書の作成expert

証拠説明書は、甲号証（証拠）の一覧表です。各証拠の番号、標目、原本・写しの別、作成日、作成者、立証趣旨を記載します。

記載項目

証拠番号 — 甲1、甲2、甲3...と連番
標目 — 証拠の名称（例: DVDパッケージ画像）
原本・写しの別 — 原本 or 写し or 電磁的記録
作成日 — 証拠が作成された日付
作成者 — 誰が作成したか
立証趣旨 — 何を証明するための証拠か

作成手順

  1. 甲号証を整理 — すべての証拠を番号順に並べる
  2. 書類作成画面を開く — lnmake.htmlの「証拠を集める」タブ
  3. 「証拠説明書を生成」ボタンをクリック
  4. プレビュー確認 — 内容に誤りがないか確認
  5. PDF出力またはMarkdown出力 — 必要な形式で保存
  6. 印刷 — A4サイズで印刷し、押印
注意点
  • 甲号証の番号は絶対に飛ばさない（甲1→甲2→甲3...）
  • 立証趣旨は具体的に書く（「申立人が著作権者であること」等）
  • 日付は令和表記を使用
  • 作成日が不明な場合は「日付不明」と記載

書類作成画面で自動生成できます。→ 書類作成画面へ

2.7KK（DB記録印刷書面）の集め方expert

KK（DB記録印刷書面）は、申立人の社内データベースから、違法アップロード動画の記録を印刷した書面です。

KKに含まれる情報

  • 動画タイトル — 侵害された著作物のタイトル
  • URL — 違法アップロードされた動画のURL
  • 発見日時 — いつ発見されたか
  • IPアドレス — アップロード者のIP
  • Torrent情報 — ハッシュ値、ピア情報等

集め方

  1. 申立人に依頼 — 該当LNのKKデータ抽出を依頼
  2. CSVまたはPDFで受領 — GoogleDrive経由が一般的
  3. 内容確認 — IPアドレス、日時が正しいか確認
  4. 印刷 — A4サイズで印刷（両面可）
  5. ページ番号を振る — 「1/10」「2/10」...と記載
注意点
  • 申立人によってはKK不要の場合あり（弁護士に確認）
  • 個人情報が含まれる場合は取り扱いに注意
  • 印刷時に文字が切れないよう余白を確保
  • 大量ページの場合はPDF化も検討

KKの要否は申立人・弁護士により異なります。LN一覧の「KK状況」列で確認してください。

2.8pl（証拠PDF）の確認expert

pl（証拠PDF）は、甲号証（甲1〜甲13等）をまとめた1つの大きなPDFファイルです。自動生成されます。

plに含まれる証拠

  • 甲1 — USB著作物（動画ファイル）のスクリーンショット
  • 甲2〜甲9 — 違法アップロード動画のスクリーンショット
  • 甲10 — 再生試験報告書
  • 甲11 — プロバイダへの照会結果
  • 甲12 — IPアドレス一覧
  • 甲13 — DVDパッケージ画像
  • 甲14 — 著作権証明書類

確認手順

  1. GoogleDriveで開く — 各LNのDriveフォルダに自動アップロードされる
  2. ページ数を確認 — 欠落がないか
  3. 画質を確認 — 文字が読めるか、画像が鮮明か
  4. 順番を確認 — 甲1→甲2→...と正しい順序か
  5. ファイルサイズを確認 — 異常に大きい・小さい場合は要確認
エラーが発生した場合
  • 「pl処理中」のままの場合 → 自動処理の完了を待つ（通常1〜2時間）
  • ページ欠落 → expertに報告、plリセットを依頼
  • 画質不良 → 元画像の確認、再生成を検討

plの生成状況はLN一覧の「pl状況」列で確認できます。→ ポータルへ

3.LN確認（beginner）

3.1LN確認の手順beginner

初めての方でも大丈夫！今日やるLNを確認して、問題なければ報告するだけです。

  1. ポータルの「ホーム」を開いて、ステップナビの案内に従う
  2. 「LN一覧」タブで「今日やること」のLNを確認
  3. LNをクリックすると詳細が開く
  4. チェックリストを見て、全部✓か確認
  5. CSVとDriveの中身を確認。ファイルが揃っていればOK
  6. 問題なければ「確認完了しました」で報告

不安な点や×があれば、expertに相談してください。
→ ポータルのLN一覧へ

3.2CSV・Drive確認手順beginner

CSV: 申立書に使うデータファイル。「CSVダウンロード」ボタンからダウンロードできればOK。

Drive: そのLN用のフォルダがあり、中にPDFが入っていれば問題なし。

何が入っているかわからなくても大丈夫。フォルダがあってファイルがいくつかあれば、expertに「確認しました」と報告してOKです。

3.3ポータルの画面の見方beginner

  • マニュアル — このページ
  • 実績・レベル — レベルやポイント
  • フィードバック — 気づいたことを報告

右上のユーザー切替で、ロール別の画面の違いも体験できます（8人登録）。
